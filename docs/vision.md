# JARVIS Core — Vision Document

## 缘起

从对话历史可以看出，这个项目经历了多次迭代和升维：

1. **Forge AI 工具工厂** — AI 自动生成 AI 工具的工业级平台
2. **Prometheus 自主 AI 系统** — 五层架构的自进化 AI
3. **JARVIS Core** — 最终的终局形态

每一步都在回答同一个问题：**如果贾维斯真的存在，它应该是什么样子？**

## 答案

贾维斯不应该是另一个工具。它是你数字生命的操作系统。

### 不局限于一个领域

贾维斯同时管理 Tony Stark 的：
- 豪宅（Home 领域 — IoT、安全、环境）
- 实验室（Engineering 领域 — 代码、设计、仿真）
- 战衣（Security 领域 — 武器系统、威胁检测）
- 公司（Finance 领域 — 投资、预算、税务）
- 社交（Personal 领域 — 日程、通讯、笔记）
- 健康（Health 领域 — 生命体征、医疗分析）

这不是几个独立应用的拼凑，而是**一个智能体同时运作在所有这些领域**。

### 关键设计决策

| 决策 | 传统做法 | JARVIS 做法 |
|------|---------|-------------|
| 领域扩展 | 写插件 | 热插拔领域模块 — 像装新肢体 |
| 记忆 | 上下文窗口 + RAG | 四级记忆：工作/情景/语义/过程 |
| 进化 | 人工调 prompt | 闭环自进化：执行→观察→优化→重复 |
| 代码执行 | 本地直接跑 | 三层沙盒隔离 |
| 多领域协作 | 串行调用 | 意图融合 → 领域路由 → 并行执行 |

### 与现有 AI 助手的区别

- vs ChatGPT: 不是对话工具，是行动系统
- vs Copilot: 不只写代码，管理全部数字生活
- vs Siri/Alexa: 不限于预定领域，无限扩展
- vs AutoGPT/BabyAGI: 不是任务链，是持续运行的智能操作系统

## 路线图

### MVP (当前)
- 8 领域骨架 + 基础路由
- 记忆引擎 + 压缩
- 进化引擎 + 优化循环
- API 服务 (REST + WebSocket)
- Docker 沙盒

### V1 — 可用
- 各领域实际功能实现
- LLM 集成 (多模型路由)
- 端到端测试覆盖
- 生产部署配置

### V2 — 智能
- 跨领域推理 ("为健康项目准备财务报告")
- 主动建议 ("您今天的日程有空档，要不要去健身？")
- 个性化学习 (适应用户风格)

### V3 — 自主
- 定时主动任务
- 目标分解
- 多实例协作
- 自托管基础设施

### V4 — 无处不在
- 边缘部署 (树莓派、眼镜)
- 完整 IoT 编排
- AR/VR 集成
- 生物识别集成

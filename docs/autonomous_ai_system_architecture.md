# 自进化自主 AI 系统 — 完整架构设计方案

> 代号：Prometheus（普罗米修斯）

## 一、系统愿景

构建一个能够**自我运行、自我进化、自我维持**的自主 AI 智能体系统。系统具有以下核心特性：

| 能力维度 | 目标 |
|---------|------|
| **自主运行** | 7×24 无间断运行，自行调度任务、管理资源、处理异常 |
| **自我进化** | 从经验中学习，自动优化提示词/工具/策略，持续提升能力 |
| **多应用互联** | 通过 MCP 协议接入任意外部工具和服务 |
| **自主检索** | 根据目标主动搜索网络，提取、归纳、存储知识 |
| **自我维持** | 通过内容创作和自媒体运营覆盖运行成本 |
| **安全合规** | 在沙盒环境中运行，所有破坏性操作需人工审批 |

## 三、核心子系统详解

### 3.1 控制平面（Control Plane）

控制平面是整个系统的"大脑皮层"，负责宏观决策和治理。

#### 3.1.1 目标管理器（Goals Manager）

```
目标层级结构：
  L0 使命层  → "通过内容创作实现自我维持"
  L1 战略层  → "建立科技类自媒体矩阵，月收入覆盖 API 成本"
  L2 战术层  → "本周发布 5 篇 AI 领域深度文章 + 3 个短视频"
  L3 执行层  → "今天搜索 DeepSeek 最新动态 → 撰写分析 → 发布到 3 平台"

目标生命周期：
  定义 → 分解 → 调度 → 执行 → 评估 → 调整/进化
```

**实现**：基于 GOAP（Goal-Oriented Action Planning）的层次化目标系统。每一层目标可以分解为下层的子目标，形成执行树。系统根据资源消耗、成功率、外部反馈动态调整优先级。

#### 3.1.2 任务调度器（Scheduler）

```
调度策略：
  ├── 定时任务（Cron）：每日热点搜索、定期发布、日志轮转
  ├── 事件驱动：搜索热点触发创作任务、用户评论触发互动任务
  ├── 优先级队列：紧急 > 高 > 中 > 低
  └── 资源感知：根据 API 额度、算力余量、时间窗口动态调度

并发控制：
  ├── 创作任务可并发（不同主题的文章并行生成）
  ├── 发布任务串行（同一平台避免风控）
  └── 学习任务后台执行（低优先级，空闲时运行）
```

#### 3.1.3 安全审计器（Auditor）

参考 Codex CLI 的 Exec Policy 机制：

```
操作风险等级：
  🔴 禁止：修改系统文件、执行未审查代码、转账/支付
  🟡 审批：文件写入（非工作目录）、新增平台账号、修改配置
  🟢 自动：读取文件、搜索网页、生成内容、发布草稿（预审核）

沙盒策略（借鉴 Codex CLI）：
  Level 1: workspace-write → 仅工作目录可写（默认）
  Level 2: read-only → 全系统只读
  Level 3: full-disk-write-access → 需审批
  Level 4: danger-full-access → 禁止
```

### 3.2 执行引擎（Execution Engine）

核心灵感：OpenAI Codex CLI 的 Agent Loop + Hermes Agent 的闭环学习。

#### 3.2.1 思考循环（Think Loop）

```
每一轮执行周期：
  ┌─────────────────────────────────────────┐
  │ 1. 感知 (Perceive)                       │
  │    ├── 读取当前目标队列                   │
  │    ├── 检索相关记忆 (Memory Retrieval)    │
  │    ├── 获取环境状态 (时间、系统资源、配额)  │
  │    └── 收集外部信号 (热点、消息、反馈)      │
  ├─────────────────────────────────────────┤
  │ 2. 推理 (Reason)                         │
  │    ├── 上下文组装 (P0-P4 Token 预算)      │
  │    ├── 模型推理 (gpt-5.1/gemini-2.5)      │
  │    └── 生成行动计划 + 工具调用             │
  ├─────────────────────────────────────────┤
  │ 3. 行动 (Act)                            │
  │    ├── 安全审查 (Auditor 检查)            │
  │    ├── 工具调度 (Tool Router)             │
  │    └── 执行结果捕获                       │
  ├─────────────────────────────────────────┤
  │ 4. 反思 (Reflect)                        │
  │    ├── 结果评估 (成功/失败/部分成功)       │
  │    ├── 错误分析 (根本原因)                │
  │    ├── 记忆更新 (存储经验)                │
  │    └── 策略调整建议 (反馈给进化引擎)       │
  └─────────────────────────────────────────┘
```

**关键设计**：每个周期的反思结果都会写入 `reflections/` 目录，进化引擎定期扫描并据此优化。

#### 3.2.2 记忆系统（Memory System）

借鉴 Novel Studio L0 结构化正典 + Hermes Agent MEMORY.md + Codex CLI 两阶段记忆管线：

```
记忆层级：
  ┌─────────────────────────────────────┐
  │ L0 工作记忆 (Working Memory)         │
  │ 当前任务上下文，生命周期 = 单次执行    │
  │ 容量：128K tokens                    │
  ├─────────────────────────────────────┤
  │ L1 短期记忆 (Short-term)             │
  │ 最近 N 轮执行的历史轨迹              │
  │ 存储：SQLite，生命周期 = 24小时       │
  ├─────────────────────────────────────┤
  │ L2 长期记忆 (Long-term)              │
  │ 结构化事实表 + 向量索引              │
  │ 存储：JSONL + ChromaDB              │
  ├─────────────────────────────────────┤
  │ L3 技能记忆 (Skill Memory)           │
  │ 可复用的执行模板和操作手册            │
  │ 存储：Markdown + YAML               │
  ├─────────────────────────────────────┤
  │ L4 元记忆 (Meta Memory)              │
  │ 关于学习过程本身的记忆                │
  │ "上次这种策略效果不好，这次换一个"     │
  │ 存储：SQLite + 版本化 Git 仓库        │
  └─────────────────────────────────────┘

记忆提取管线（借鉴 Codex CLI 两阶段）：
  Phase 1 — 提取：后台模型扫描执行日志，提取关键事实
  Phase 2 — 整合：主模型去重、合并、更新结构化正典
```

### 3.3 进化引擎（Evolution Engine）

#### 3.3.1 进化什么（What to evolve）

| 层级 | 进化对象 | 方法 | 灵感来源 |
|------|---------|------|---------|
| L1 | 提示词模板 | TextGrad 文本梯度优化 | Nature 2025 |
| L2 | 工具集 | AgentFactory 技能生成+测试+注册 | arXiv:2603.18000 |
| L3 | 执行策略 | AFlow 工作流自动搜索 | ADAS 算法 |
| L4 | 记忆结构 | 自动摘要/索引/剪枝 | Hermes Agent |
| L5 | 模型适配器 | LoRA 微调（远期） | Online RL |

#### 3.3.2 何时进化（When to evolve）

```
触发条件：
  ├── 即时：单轮执行失败 → 立即反思 + 提示词微调
  ├── 定时：每 6 小时 → 扫描近期失败模式，生成优化建议
  ├── 阈值：连续 N 次同类失败 → 触发工具/策略重构
  └── 周期性：每周 → 全面审计 + 提示词典雅化 + 技能淘汰

安全护栏：
  ✗ 不准跨过 Auditor 直接修改敏感配置
  ✗ 不准在未经测试的情况下替换核心工具
  ✓ 所有更改记录版本历史，支持一键回滚
  ✓ 重大变更需人工审批
```

#### 3.3.3 如何进化（How to evolve）

```
进化流水线：
  收集轨迹日志 → 离线分析 → 生成候选改进 → 并行测试 → 选出最优 → 灰度上线 → 全量

TextGrad 提示词优化示例：
  1. 记录：使用 Prompt_A 生成文章，读者反馈 AI 腔明显
  2. 分析：LLM 评估失败原因 → "提示词中缺少'避免高频连接词'约束"
  3. 生成候选：Prompt_A' = Prompt_A + "禁止使用'然而''总之''在这片土地上'"
  4. 测试：A/B 测试 10 篇文章 → Prompt_A' 的"AI 腔得分"降低 40%
  5. 采纳：Prompt_A → Prompt_A'，版本号 1.0→1.1
```

## 四、能力层（Capability Layer）

### 4.1 创作引擎（Creator Engine）

```
创作流水线：
  选题发现 → 深度检索 → 大纲生成 → 正文撰写 → 质量审查 → 多版本适配 → 封面生成

支持的输出格式：
  图文：公众号文章 / 知乎回答 / 小红书笔记 / 头条文章
  视频脚本：抖音 60s / B站 5-10min / YouTube 10-15min
  音频：播客脚本 → TTS 合成
  多语言：中文 → 英文/日文（HY-MT 2 翻译）

质量保障：
  ├── AI 腔检测（10 个高频词汇频率监控）
  ├── 事实核查（引用源回溯验证）
  ├── 原创度检测（与已有内容库比对）
  └── 可读性评分（Flesch-Kincaid 中文适配版）
```

### 4.2 搜索引擎（Search Engine）

```
自主搜索策略：
  L1 快速搜索：web_search → 标题+摘要 → 判断相关性
  L2 深度抓取：web_fetch → 全文提取 → 结构化归纳
  L3 对比验证：多源交叉验证 → 矛盾检测 → 置信度评分
  L4 知识沉淀：新事实 → 事实表 → 向量索引

触发条件：
  ├── 定时触发：每小时扫描科技/AI 热点
  ├── 目标驱动：创作任务需要背景调研
  ├── 好奇心驱动：发现新概念/工具 → 主动深入了解
  └── 用户指令：直接搜索请求
```

### 4.3 发布引擎（Publish Engine）

借鉴 AiToEarn 的多平台 Adapter 架构：

```
支持平台（首批）：
  国内：微信公众号 / 知乎 / 小红书 / 抖音 / B站 / 头条号
  海外：Medium / Substack / X(Twitter) / LinkedIn

发布策略：
  ├── Adapter 模式：每个平台独立封装，统一接口
  ├── 双路径：有 API 用 API，无 API 用 Playwright 浏览器自动化
  ├── 定时发布：模拟人类作息，避开凌晨发布
  └── 互动管理：自动回复常见问题，标记高价值评论
```

### 4.4 变现引擎（Monetize Engine）

```
变现路径（按优先级）：
  P0 自媒体广告分成：
    ├── 公众号流量主 / 头条号广告
    ├── YouTube AdSense
    └── Medium Partner Program

  P1 内容付费：
    ├── 知乎盐选 / 付费咨询
    ├── 小报童 / 知识星球
    └── Substack 付费订阅

  P2 品牌合作（远期）：
    ├── CPS 带货（好物推荐）
    ├── 软文合作
    └── AI 工具推荐返佣

  P3 服务变现（远期）：
    ├── AI 写作代运营
    ├── 技术咨询
    └── 定制报告
```

### 4.5 学习引擎（Learner Engine）

```
学习路径：
  ├── 被动学习：从每次执行结果中提取经验
  ├── 主动学习：空闲时阅读技术文章/论文
  ├── 课程学习：从简单任务逐步进阶到复杂任务
  └── 反思学习：定期回顾近期失败，总结教训

学习产出：
  ├── 新技能（Markdown 操作手册）
  ├── 更新知识库（ChromaDB）
  ├── 优化提示词模板
  └── 更新工具集
```

## 七、安全与合规

### 7.1 安全体系

```
多层防护：
  Layer 1: 沙盒隔离（Docker + Codex CLI 风格 exec policy）
  Layer 2: 权限分级（只读 / workspace-write / 审批 / 禁止）
  Layer 3: 审计日志（所有操作可追溯）
  Layer 4: 人工审批（高风险操作强制执行）
  Layer 5: 回滚机制（配置版本化，一键恢复）
```

### 7.2 合规清单

| 领域 | 措施 |
|------|------|
| **内容合规** | AI 生成内容标注，不涉政/不涉黄/不涉暴 |
| **版权保护** | 引用标注来源，原创度检测，避免洗稿 |
| **平台规则** | 遵守各平台自动化发布规范，模拟人类行为频率 |
| **数据隐私** | 本地存储优先，不泄露用户数据 |
| **AI 标识** | 发布内容标注"AI 辅助创作" |
| **法律底线** | 不生成虚假信息，不操纵舆论，不侵犯他人权益 |
